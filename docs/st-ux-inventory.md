# SillyTavern 界面行为清单（草稿 · 2026-07-08）

> 用途：梨园 Web UI 的设计输入（PLAN-PHASE3 §6）。**学行为，不学实现**——本清单来自对 ST 的使用观察与社区常识，不参考其源码（D5 AGPL 纪律）。
> 请以重度玩家身份逐条补正：改标注、删错漏、补我没见过的用法。
> 标注含义：[M1] 本期采纳 · [M2] 会话循环期 · [M3] 旁侧产物期 · [M4+] 素材/氛围期 · [不学] 哑管道补偿品或过度复杂

## 一、布局骨架

- [M4+] 中央聊天流常驻 + 左右滑出面板（左：角色与聊天管理；右：世界书/设置）；手机上是全屏抽屉
- [M3] 顶栏精简 + 抽屉化收纳（ST 顶栏图标过密，学收纳不学密度）
- [M4+] **全屏背景图 + 半透明气泡浮层**——氛围体验的核心，RP 用户换背景如换舞台
- [M1] 输入区固定底部：文本框 + 发送/停止 + 溢出菜单（后续挂重生成/继续等）
- [M1] 消息区宽度上限、桌面居中窄栏（✅ M0 已做）

## 二、交互词汇

- [M2] **Swipe 重 roll**：最后一条 AI 消息左右箭头/滑动手势，计数「2/5」——ST 最高频操作，梨园对应会话树兄弟分支（pi 原生支持，比 ST 的 swipes 数组更强）
- [M2] 消息操作条（hover/长按）：编辑、复制、删除、从此处开分支、重生成
- [M2] 消息就地编辑（textarea + 确认/取消）——注意 D10 边界：编辑的是**用户自己会话存档**，属用户主权，允许
- [M2] 「继续说」（continue）：让主演接着上一条往下写
- [M1] 生成中停止按钮替换发送按钮（✅ M0 已做）
- [M1] 底部跟随 + 离底时「↓」跳底按钮
- [M4+] 上滑惰性加载更早消息（长会话性能）
- [M3] `/` 命令自动补全弹条
- [不学] 消息「隐藏」（从上下文排除的眼睛图标）——上下文取舍在梨园是 D9/导演的职责，不该让用户手工管理上下文

## 三、排版审美

- [M1] "对白"着色、*动作*斜体（✅ M0 已做，社区默认审美）
- [M1] 楼层号 #N（社区文化：「第 N 楼」）
- [M1] 思考过程折叠块（ST 的 reasoning 折叠同款体验）
- [M2] Markdown 全量渲染（表格/列表/代码/引用）——注意：卡作者用表格画状态栏的习惯不鼓励（状态归面板），但渲染能力要有
- [M4+] 头像（用户/角色，圆方可选）；角色头像即卡面 PNG
- [M4+] 字号/行距设置项
- [不学] 消息内嵌 HTML/脚本执行（ST 允许卡作者塞 HTML/JS——安全面与 D10 双重反对；纯 markdown 足够）

## 四、氛围系统

- [M4+] 每聊天背景图 + 暗化遮罩（保可读性）
- [M4+] 主题 = CSS 变量集（现有暗色即默认主题；社区主题文化可承接）
- [M4+] 立绘/表情差分（expressions）：角色情绪立绘随剧情切换——与「agent 持有前端操作权」天然契合（scribe 旁侧判定情绪 → 前端换立绘，产出是数据非正文，D10 合规），是梨园能做得比 ST 更聪明的点
- [不学] 打字音效（小众，后台可扩展位留白即可）

## 五、ST 有而梨园刻意不同的

- 状态栏：ST 靠模型在正文里画 → 梨园状态归系统面板（M3），正文归模型（D10）
- token 计数器/上下文进度条：ST 用户被迫关心上下文 → 梨园 D9+compaction 后台处理；M3 状态面板给一行简洁会话统计即可
- Prompt 管理器/正则脚本/变量 UI：装配逻辑即 harness 代码（D4），不做 UI
- swipes 数组 → 会话树：重 roll 不丢历史，任何节点可回溯（M2 的树导航 UI 是 ST 做不到的）
