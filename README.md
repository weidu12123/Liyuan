# 梨园 Liyuan

**以 AI 为主体的角色扮演应用。** 不是给聊天软件接上模型，而是让一个完整的 agent 登台演戏——它有记忆的章法、停笔问你的分寸、亲手画图立牌的能力，和一个随剧情回档的世界。

## 梨园是什么

传统的 AI 角色扮演，本质是一问一答的哑管道：前端把提示词拼好、发给模型、显示回复。模型每轮只有一次开口机会，记不住的靠玩家复读，看不见的靠玩家脑补，不满意的只能事后重 roll。社区多年积累的正则脚本、状态栏模板、预设技巧，大多是在补偿这个架构的先天缺陷。

梨园换了地基：**模型是一个真正的 agent**。落笔之前，它可以查设定、核对账本、翻你上传的素材；剧情走到岔路口，它会停下来问你；世界需要一张地图，它自己动手画。补偿品所服务的那些问题，在这个架构里大多不再存在。

由此带来一个根本差异：**传统前端的功能上限由开发者和插件作者决定；梨园的功能上限由模型的智能决定。** 模型越强，梨园越强，不用等任何人更新。

## 七件事，传统 AI 角色扮演做不到

### 1. 记忆 —— 为叙事重新设计的上下文架构

角色扮演最重要的是记忆，而模型的记忆就是上下文窗口——稀缺资源。agent 每轮的工具调用、中间过程若原样堆进历史，窗口很快被过程垃圾填满，更糟的是提前触发有损压缩，被压掉的恰恰是剧情。

梨园在架构层重新剪辑每一轮上下文：会话文件是完整素材（全量存档、可回放），真正送给模型的是**只保留叙事正文的成片**——过程性内容在代码层被确定性裁掉，实测每轮节省 53%–63%。同样的窗口，有效剧情容量翻倍以上；同样的模型，拿到一份纯净的剧情稿而非施工现场，注意力不被稀释，智力发挥也更完整。

裁剪只是第一层。梨园的记忆是四层结构：

| 层 | 机制 | 承载什么 |
|---|---|---|
| 纯净上下文 | 每轮自动裁剪过程、保留成果 | 工作记忆：最近剧情的原文 |
| 结构化账本 | 旁侧模型每轮自动记账，快照随剧情走 | 事实记忆：物品、好感、时间、伏笔 |
| 检索资产 | 世界书 + 知识库，agent 主动查询 | 设定记忆：海量设定不占窗口，用时才取 |
| 剧情化压缩 | 按叙事逻辑生成前情提要（人物/承诺/事实账） | 超长程：即使压缩，也压得懂剧情 |

事实不依赖模型的脑内记忆——模型忘了，账本没忘，而且账本每轮都在它眼前。隔几十轮追问一件物品的下落、压缩之后伏笔依然回收，靠的是结构，不是运气。

> 窗口里只有剧情，事实进账本，设定进检索库。同样的模型、同样的窗口，记忆上限和智力发挥都不在一个量级。

### 2. 共创剧情 —— 从拆盲盒到坐上导演椅

传统角色扮演里，你对剧情的全部干预都是事后的：模型落笔那一刻走向已定，不满意只能重 roll、再 roll——拆盲盒。重要角色被随手写死，剧情拐进你不想去的方向，你只有否决权，没有参与权。

梨园把 agent 产品的「权限门禁」翻译成了「剧情门禁」：

- **决策卡**：重要新角色如何定型、关键设定定死、难以回头的重大转折（死亡、背叛、关系质变）——agent 会停笔，在叙事流里摊开选择卡：它构思的几个方向、一个自由输入框（你自己写走向）、一个收笔按钮（本回合中断，笔还给你）。选完，剧情按你定的走。卡片留痕在叙事流里，日后回看，每个岔路口都清晰可见。
- **询问 / 静默两档**：想深度共创开询问档；想纯沉浸开静默档，回到完全不打扰的传统体验。会不会问个不停？不会——日常剧情、场景细节、一次性路人明确不问，「问烦了比不问更糟」是设计原则。
- **随时插话，不破坏剧情**：`//` 或 `((` 开头就是场外发言——不占剧情楼层、不进剧情记忆。你可以中途和 agent 商量走向、查设定、改状态、下任何指令，说完回戏。戏是戏，话是话。

这套「该问必问」不是靠提示词祈祷：写入世界观、知识库这类落盘动作有代码层的硬门禁——你不点头，它物理上写不进正典；正文里先斩后奏的重大转折则有旁侧审计事后告警，并建议一键回档。

> 传统角色扮演里你是观众，最多把不满意的一集扔回去重拍；在梨园你坐在导演椅上——每个重大岔路口，笔会停下来问你。

### 3. 可视化 —— agent 亲手搭建的面板

传统角色扮演是纯文字流，血条、背包、地图全靠脑补；即便有「状态栏」类玩法，也是作者预先写好模板、再祈祷模型每轮照着画——预置了什么，你才有什么。

梨园把前端操作权交给 agent。**面板不由开发者穷举，由 agent 按剧情需要现场发明**：

- 跑团开打，它自己建一间装备库，捡了铁剑实时入库；
- 进了迷宫，它手画一张 SVG 地图，走一步更新一步；
- 悬疑本，它立一块线索板，新证据随剧情钉上去。

三个硬设计：**面板随世界回档**——回档之后，地图和装备库跟剧情一起回到过去，它们不是装饰，是世界状态的一部分；**沙箱渲染**——SVG/HTML 一律在无脚本沙箱里显示，所以面板可以放心导入导出、社区分享，藏了恶意代码也执行不了；**正文归正文**——面板永不挤占叙事流，正文永远是干净的小说。

插图、语音也在同一哲学下：生图的提示词由 agent 从剧情和账本亲自构造——它知道「她穿的是三章前你送的月白裙」。

> 传统角色扮演的一切都发生在想象里；梨园的 agent 会自己动手——开一间装备库、画一张地图、立一块线索板，而且它们随剧情回档一起回到过去。

### 4. 扩展 —— 用中文写插件，agent 自己学本事

传统前端的扩展是程序员的事：想加功能要写代码插件，普通玩家只能等社区施舍；而且插件是死代码，不理解剧情。

梨园的扩展性直接继承自 agent 世界的标准件：

- **技能（Skill）**：一段自然语言写的说明书就是一个插件。想要剧情配图？技能里写一句「剧情出现新场景时，调用某某接口，提示词按当前剧情组织」——不写一行代码，而且图天然符合上下文，因为提示词是 agent 亲自从剧情里组的。语音播报同理，一句话教会它何时开口。
- **agent 自写技能**：把 agent 指向一个它从没见过的服务，它自己摸接口、试参数、跑通之后**把调用方法写成技能笔记**——下次全新会话零提示、一步直达。工具的说明书由工具的使用者自己写、自己攒，技能文件还能分享给别人。
- **MCP 外设**：标准协议接入现成的外部工具生态（浏览器、搜索、数据库、记忆库……）；自动识别你机器上已有的 MCP 配置，默认全关、按对话启用，用不到的不打扰。

探索空间由你定义：接一个向量数据库 MCP 给超长剧情做检索记忆、接浏览器让 agent 查资料入戏——这些是等你发掘的玩法方向，不是开发者预设的功能清单。

> 传统的插件要用代码写；梨园的插件用中文写——一段话就是一个技能，而且 agent 自己学会的本事，还会自己记成笔记留给下次。

### 5. 世界线 —— 存档存的是整个世界

想体验同一张卡的不同结局，传统做法只有重开（一切归零）或翻旧记录改写（状态全乱）；即便有分支功能，分支的也只是聊天记录的文字。

梨园给的是游戏级的存档语义：

- `/store` 钉存档，`/back` 回档，`/line` 查看世界线全景；
- **回档回的是整个世界**：剧情正文、状态账本（物品/好感/时间）、agent 建的面板（地图/装备库）、知识库挂载——全部一致地回到那个时间点。桌上的信物从正文和账本里同步消失，地图回到你还没探开的样子；
- **世界线的划分很克制**：回档后走出与旧后续不同的路、再次存档时，才自动分出新世界线——不是每按一次就炸出一条线。重 roll 变体与世界线无关，随便试；
- 所有历史永不丢失：旧世界线永远在那里，随时回去续写；
- **存档是零损耗的**：即使经历过上下文压缩，回到压缩前的存档，上下文自动恢复为未压缩的全量原文——压缩留在旧分支上，不跟过来。每条世界线从干净的素材重新出发。

> 游戏存档存的是世界状态，不是对话日志。梨园的回档，剧情、账本、地图、装备库一起回到那个时刻——然后同一张卡，你可以活出完全不同的结局。

### 6. 知识库 —— 世界书是别人写好的，知识库是你玩出来的

游玩产生的知识在传统前端里无处安放：剧情中诞生的精彩设定——agent 即兴发明的城市、你们一起定下的魔法规则——散落在聊天记录里，换卡、开新会话就死了。「玩」和「积累」是断开的。

梨园的知识库是跨对话、跨角色卡的活资产：

- **建库挂载**：面板点一下，或一句话让 agent 建；一个库可挂到任意对话，多库并挂；
- **双向共创**：你可以亲手写入；agent 在剧情推进中遇到值得留的东西（新设定、重要物品、世界规则）会主动提议入库；
- **入库有门禁**：agent 想写，先弹批准卡——明示「跨对话共享，永久保留」，你点头才成为正典；
- **挂上即活**：库条目并入检索，agent 主动查得到，关键词也能被动命中；挂载关系随世界线走，回档一起回退；
- **出口自由**：一键导出为 SillyTavern 兼容的世界书格式，你的积累可以分享、可以带走。

与世界书的分工一句话：跟着**本剧本**走的设定进随卡的补充设定集，**跨剧本**通用的知识进知识库——agent 也被教了同样的分工规则。

> 剧情里长出的新设定，agent 主动提议入库，你点头才成为正典——跨对话、跨角色卡积累。你的世界，终于可以越玩越厚。

### 7. 素材库 —— 给 agent 一个文件夹和一双眼睛

传统前端里，图片和文件是一次性的：发出去看一眼就沉底。想让 AI 参照你的人设图、设定文档、地图，要么贴成大段文字，要么做不到。

梨园的上传区是**创作素材库**，而它的工作方式与众不同：**别处把文件塞给模型，梨园让 agent 自己去拿。**

- 上传只是落盘——**内容不进上下文，消息里只附一行路径**；agent 每轮都知道素材库里有什么，什么时候看、看多少，是它的判断；
- 读过的内容轮次结束即从上下文清出，需要时随时再读——素材在盘上，不在记忆里，传一百个文件窗口也不涨（这正是第 1 条记忆哲学的又一次显形）；
- 人设图扔进去让它照着描写外貌，设定文档扔进去一句「读完再开场」它就入戏，还能让它把素材整理进世界书或知识库；
- 素材跨会话、跨角色卡复用，「附到消息」不重复上传——传一次，处处能用；
- 体验零门槛：按钮多选、拖拽、粘贴剪贴板图片都行，图片直接显示在对话里。

> 梨园没有发明「附件系统」——它把文件放到 agent 的手边，剩下的交给 agent 自己。文件不是聊天的附件，是你世界的素材。

## 从 SillyTavern 搬家

SillyTavern 社区多年积累的角色卡与世界书，是 AI 角色扮演的共同财富。梨园完整承接这份生态：**你的收藏不需要重建。**

- ✅ **角色卡**：PNG 卡（V2/V3 内嵌格式）与 JSON 卡直接导入，含卡内嵌世界书；
- ✅ **世界书**：JSON 直接导入，constant / 关键词激活语义保留，支持多本挂载。而且是**双向**的——梨园知识库可导出回 SillyTavern 世界书格式，你的数据进出都自由；
- ✅ **聊天记录**：jsonl 旧档可导入续玩——自动清洗（剥离旧状态栏与思维链，避免带坏文风），旧剧情自动摘要、自动建账；
- ⚠️ **预设**：提供转换器（`node scripts/convert-preset.mjs <ST预设.json>`），提示词内容块搬入对应位置、采样参数照常生效。但如实说：预设是为一问一答的旧架构设计的，其中相当一部分块（思维链模板、状态栏指令、装配技巧）本质是在补偿旧架构的缺陷——这些问题在梨园已被架构本身解决，对应的块失去了作用对象。预设能带进来，效果请自行实测；你可能会发现大部分块已经不需要了；
- ❌ **明确不兼容**：正则脚本、STscript、前端插件、角色卡自带的 HTML 界面——这些不是数据，是绑定旧架构的玩法，不在承接范围。

> 梨园未使用任何 SillyTavern 代码，全部格式解析按公开规范独立实现。

## 快速开始

前置：**Node.js ≥ 22**，任一 OpenAI 兼容 API Key（如 DeepSeek）。

```bash
cd Liyuan   # 若仓库根即本目录则跳过

# 首次：从示例生成本地配置（勿把填了 Key 的文件提交 git / 发给别人）
cp liyuan.agent.example.json liyuan.agent.json
cp liyuan.config.example.json liyuan.config.json
# 编辑 liyuan.agent.json → 填入 apiKey 与模型 id

npm install
npm run web:build   # 首次或前端改动后（已有 web/dist 可跳过）
npm run web         # 起服务；npm run web:new 开新会话
```

- Windows：双击 `start.bat`
- Linux / macOS：`chmod +x start.sh && ./start.sh`
- 浏览器打开控制台打印的地址（默认 `http://127.0.0.1:7620`）；控制台同时打印局域网地址，手机连同一 Wi-Fi 直接访问
- 前端开发：`npm run web:dev`（Vite 热更新，`/ws` 自动代理到 7620）

### 服务器部署（Linux / VPS）

三种方式任选，细节见 [deploy/README.md](deploy/README.md)：

```bash
# 方式 A：一键安装脚本（装成 systemd 服务，自动安装 Node 22）
curl -fsSL https://raw.githubusercontent.com/weidu12123/Liyuan/v1.0.0/deploy/install.sh | bash
# 装好后填 Key：nano /opt/liyuan/liyuan.agent.json，然后 systemctl restart liyuan

# 方式 B：Docker Compose（数据全在命名卷里，升级重建不丢档）
git clone --depth 1 https://github.com/weidu12123/Liyuan.git && cd Liyuan
docker compose up -d --build

# 方式 C：手动打包上传（Windows 本机执行，产出免 node_modules 的 zip）
powershell -File scripts/pack-for-linux.ps1
```

**请勿把 7620 端口裸暴露公网**——服务本身无鉴权，对外请套反向代理 + 鉴权。

配置文件分两份：**`liyuan.config.json`** 管角色卡 / 世界书 / 用户身份等（面板里也能改）；**`liyuan.agent.json`** 管模型与 Key（**勿提交仓库**）。旧版 `.rp-*` 目录与 `rp.config.json` 启动时自动迁移。

## 进阶

- **斜杠命令**（Web 输入框直接敲，带补全）：`/state` 账本 · `/lore` 设定检索 · `/import` 导入旧档 · `/store` `/back` `/line` 存档与世界线 · `/rewind` `/branch` 回退与分支 · `/compact` 手动压缩 等；
- **MCP**：启动时自动扫描 `~/.claude.json`、Cursor 配置、`~/.liyuan/mcp.json`、项目 `.mcp.json` / `.liyuan-mcp.json`。**默认全关**，在「扩展能力 → MCP」按对话开启；开关记为新对话默认；
- **语音（TTS）**：配置环境变量 `LIYUAN_TTS_BASE_URL` + `LIYUAN_TTS_API_KEY`（或直接用 `OPENAI_API_KEY`）后，agent 可用语音工具朗读；模型与音色用 `LIYUAN_TTS_MODEL` / `LIYUAN_TTS_VOICE` 指定；
- **技能**：`.liyuan-skills/` 下的 markdown 即技能，可让 agent 自写（详见上文第 4 条），也可手写后在「扩展能力」面板控制是否暴露给 agent。

## 已知边界（如实说）

- 不运行角色卡自带的正则脚本与独立 HTML 前端（旧生态专用玩法，见「搬家」一节）；
- 看图需要视觉模型；非视觉模型下 agent 会如实告知看不到，不会假装看见；
- 预设在 agent 架构下的效果需自行实测（原因见「搬家」一节）；
- 决策卡「问的分寸」、主动建面板与主动入库的积极性，都与所用模型的智能正相关——**模型越强，梨园越强**；
- 剧情正文永远是模型的原始输出：梨园的代码与辅助模型只做输入侧加工、结构化记账和元信息标注，绝不改写、补写正文。修音可以，假唱不行。
